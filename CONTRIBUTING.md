# Contributing

## Branch Naming
- `feat/` — new features
- `fix/` — bug fixes
- `chore/` — tooling, deps, refactors

## Pull Requests
Use the PR template. All PRs require one review and passing CI before merge.

## Commit Style
Follow Conventional Commits: `feat:`, `fix:`, `docs:`, `chore:`.
